Bash variables

$1 is the first command-line argument passed to the shell script. Also, know as Positional parameters. For example, $0, $1, $3, $4 and so on. If you run ./script.sh filename1 dir1, then:

$0 is the name of the script itself (script.sh) $1 is the first argument (filename1) $2 is the second argument (dir1) $9 is the ninth argument ${10} is the tenth argument and must be enclosed in brackets after $9. ${11} is the eleventh argument.

Bash parameter expansion

so when manipulating a variable we use curly braces basic pattern matching

name=reece
echo "$name"       # param value
echo "${#name}"    # param length
echo "${name:0:4}" # getting the substring ${paramter:offset:length}
echo "${name^}"    # title case
echo "${name^^}"   # upper case
echo "${name,}"    # first char is lower case
echo "${name,,}"   # lower case
unset name # returns variable to null
echo "${name:-Reece}" # setting the default
echo "${name:-$USER}"

#reece

Substitution, poor man’s sed in bash

freedom='GNU Not UNIX ; GNU plus Linux'

# Replace first occurrence of pattern with string
echo "${freedom/GNU/Gahnoo}"

# Replace all occurrences of pattern with string
echo "${freedom//GNU/Gahnoo}"

# Remove first match of pattern
# match every character then a space and then a semicolon and replace it with nothing
echo "${freedom/* ; /}"

freedom='GNU Not UNIX ; GNU plus Linux ; Gahnoo slash linaks ; GNU/Linux'

echo "$freedom"

# remove all occurrences of pattern
# will delete all lines
echo "${freedom//* ; /}"

# deletes last character because nothing is being replaced
# % means start at end, ? means match first character
echo "${freedom%?}"
