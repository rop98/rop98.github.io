Short notes on plantuml

Posted by Reece Petersen on Monday, January 1, 0001

plantuml

installing and running

  • sudo pacman -S graphviz plantuml
  • plantuml (produces png)

state machine syntax

[] = start node -> = coming from start node Pending = state name Pending : project = state transition name Lapsed -> [] = going to end node

@startuml
[*] -> Pending : Project
Pending --> Active : [projectstartdate = currentdate]
Active --> Lapsed : [projectfinishdate = currentdate]
Lapsed --> [*] : after [6 months]
@enduml

Class diagram

@startuml

skinparam roundcorner 20
skinparam class {
BackgroundColor #FFFFFF
ArrowColor DarkGreen
BorderColor DarkGreen
}

abstract class Person {
    #Id: string
    #name: string
    #age: int
}

class Employee {
    +Department: Department
    +Salary: Decimal
}

class Professor {
    +Expertise: String
}

class Student {
    +Grade: int
    +GradeLetter: Char
    +GraduationYear: Year
    +void UpdateGrade()
}

Person <|-- Employee
Employee <|-- Professor
Person <|-- Student

@enduml

Use case diagram

@startuml

left to right direction
skinparam packageStyle rectangle

actor Lecturer
actor Student
actor Admin

rectangle University-System {
    Lecturer --> (Grade Exam)
    Lecturer --> (Manage Course)
    Lecturer ---> (Access Course Information)
    Student ---> (Access Course Information)   
    Student ---> (Enroll in Course)
    Admin ---> (Access Course Information)
    Admin ---> (Manage Course)  
}

@enduml

@startuml

left to right direction
skinparam packageStyle rectangle

skinparam roundcorner 20
skinparam usecase {
BackgroundColor #FFFFFF
ArrowColor DarkGreen
BorderColor #5E00FF 
}


actor Customer #white;line:5E00FF;line.bold;text:black
actor System #white;line:5E00FF;line.bold;text:black

rectangle Buy_a_product {
    Customer --> (Browse catalog)
    (Browse catalog) .-down-> (Select items to buy) : includes
    Customer --> (Goes to checkout)
    Customer --> (Fills in shipping information)
    (Fills in shipping information) .-down-> (Inputs address) : includes
    (Fills in shipping information) .-down-> (Chooses next-day delivery) : extends
    (Fills in shipping information) .-down-> (Chooses 3-day delivery) : extends
    (Presents full pricing information) <--- System 
    Customer ---> (Fills in credit card information)   
    (Authorizes purchase)                <---  System 
    (Confirms sale immediately)          <---  System 
    (Sends confirming email to customer) <---  System 
}

@enduml
