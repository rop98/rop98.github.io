Syncthing and How I use it

Posted by Reece Petersen on Thursday, May 21, 2020

Overview

Syncthing is like megasync, dropbox or onedrive but using your own storage,

Syncthing is open source, syncthing can be run locally within your LAN without using global data. In order to use it with only local data, UPNP needs to be enabled on the local network. This can be set usually on 10.0.0.2 or by connecting to your router. Then in the advanced settings, UPNP will be found alongside the DLNA settings. Next in the settings for syncthing turn off the following:

  • Enable NAT Traversal
  • Global Discovery
  • Enable Relaying

You should just be left with Local Discovery.

Setting this across all devices making use of syncthing will ensure you don’t make use of Global data and you are only operating within the confines of a local network. This reduces attack surface area.

If you need more convenience it will make use of the global servers and relaying in order to find your devices and sync them. When this is in effect, the default. Ports are to be changed according to this archwiki extract.

Use a different listen address port (https://docs.syncthing.net/users/config.html#listen-addresses) for each machine, like tcp://:22010 , tcp://:22011 , tcp://:22012 and so forth. This will differentiate the nodes on the global discovery servers and avoid the “Connected to myself - should not happen” message on the other local devices whenever they leave the LAN.
